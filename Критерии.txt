1. Содержание и структура отчета:
- Полнота разделов:
  * Для прикладных проектов: введение, постановка задачи, методы решения, реализация, тестирование, результаты, выводы.
  * Для исследовательских: гипотеза, обзор литературы, методология, эксперименты, анализ данных, выводы.
- Четкость целей и задач: ясность формулировки проблемы, гипотезы, научной/практической значимости.
- Анализ существующих решений:
  * Для прикладных: обзор аналогов и технологий.
  * Для исследовательских: анализ теоретической базы, цитирование источников.
- Логичность изложения: связь между этапами, последовательность аргументации.
- Глубина проработки:
  * Для прикладных: детализация технических решений, анализ альтернатив.
  * Для исследовательских: обоснование выбора методов, валидность экспериментов.

2. Техническая реализация / Научная методология:
Для прикладных проектов:
- Качество кода/архитектуры: модульность, тестирование.
- Использование инструментов: актуальные технологии, CI/CD, системы контроля версий (Git).
- Работоспособность: логика решения, корректность прототипа (если MVP не требуется).

Для исследовательских проектов:
- Корректность методологии: воспроизводимость, контроль переменных, статистическая значимость.
- Качество экспериментов: объем данных, выбор метрик, обработка ошибок.
- Использование специализированных инструментов: научные библиотеки (например, NumPy, Pandas), ML-фреймворки (например, TensorFlow).

3. Документация:
- Техническая/Научная документация:
  * Для прикладных: описание архитектуры, API, инструкции по развертыванию.
  * Для исследовательских: детализация экспериментов (датасеты, параметры и описание алгоритмов).
- Оформление:
  * Единый стиль, грамотность, структурированность.
  * Для исследовательских: оформление по научным стандартам (ссылки, формулы).

4. Результаты и анализ:
Для прикладных проектов:
- Соответствие функциональным требованиям.
- Метрики эффективности: производительность, точность, оптимизация ресурсов.

Для исследовательских проектов:
- Валидность гипотезы: подтверждение/опровержение на основе данных.
- Научная новизна: сравнение с существующими работами.

Общее для всех типов проектов:
- Рефлексия: анализ ошибок, ограничений, предложения по улучшению.
- Практическая/теоретическая ценность: возможность внедрения или дальнейшего исследования.

Дополнительные критерии:
- Инновационность:
  * Для прикладных: нестандартное решение, оптимизация процессов.
  * Для исследовательских: новый метод или значимое улучшение существующих подходов.
- Публикационная активность: подготовка статьи или тезисов (для исследовательских).
- Командная работа: распределение ролей, использование Agile/Scrum (для командных проектов).
